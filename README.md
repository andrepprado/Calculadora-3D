Calculadora 3D – Case Study

A practical and functional case study developed to solve a real-world problem: accurately calculating 3D printing costs based on multiple variables such as material usage, machine time, and fixed production factors.

This project demonstrates how a simple interface combined with precise calculation logic can transform a manual, error-prone process into a fast, reliable, and scalable digital solution — built entirely with pure HTML, CSS, and JavaScript.

📌 About This Project

The Calculadora 3D was designed to standardize and automate cost estimation for 3D printing, eliminating inconsistencies in pricing and improving decision-making.

The application allows dynamic calculation based on:

Quantity of items
Material consumption
Print time
Additional fixed costs (e.g., packaging, finishing)

All calculations are handled in real time, ensuring clarity and immediate feedback.

🧱 Project Structure

The project follows a simple and efficient architecture:

index.html – Interface structure and input fields
style.css – Layout, responsiveness, and visual styling
script.js – Core calculation logic and dynamic updates
/img – Visual assets (if applicable)

No frameworks or external libraries were used — ensuring full control and lightweight performance.

📌 Challenge Summary

Manual cost calculation in 3D printing often leads to:

Inconsistent pricing
Incorrect scaling (unit vs. batch cost)
Misallocation of fixed costs
Lack of transparency in pricing composition

The goal was to create a reliable, reusable, and scalable calculator that:

Standardizes pricing logic
Separates fixed vs. variable costs
Avoids distortion in unit cost as quantity increases
Provides immediate and clear results
🚀 The Solution
Centralized calculation logic in JavaScript
Proper distribution of fixed costs (e.g., packaging per order, not per unit)
Dynamic recalculation based on quantity changes
Clear separation between:
Cost per unit
Total cost
Real-time UI updates without reload
🛠 Technical Implementation

The system was built with:

Pure HTML for structure
CSS for layout and responsiveness
JavaScript (ES6+) for all calculation logic

Key logic considerations:

Fixed costs applied per order (not per unit)
Variable costs scaled correctly by quantity
Prevention of illogical scenarios (e.g., unit cost increasing with volume)
Clean and maintainable function structure
